# pyrocut — frontend (apps/web)

Дашборд pyrocut: юзер за логином превращает URL лендинга в анимированное видео
(launch teaser / social reel). Лендинг готов отдельно (`landing template/`).
Бэкенд (API, БД, очередь, рендер) — отдельный промпт; мы только потребляем его
API и типы из `packages/shared`.

## Контекст
- Соло-фаундер, MVP за 3-4 дня. Простое решение, которое шипится сегодня > идеального.
- Без enterprise-паттернов без явной нужды.
- Статусы — через Supabase Realtime (подписка), не polling.

## Стек
- Next.js 15 (App Router) + TypeScript + Tailwind v4 (CSS-first `@theme`).
- Supabase JS client (auth + realtime + signed URLs).
- Иконки — `lucide-react`. Примитивы — свои (Radix только под доступность).
- Монорепо pnpm: `apps/web` (UI) + `packages/shared` (типы-контракт).

## Границы
- Владеем: `apps/web/app/**`, `apps/web/components/**`, `apps/web/lib/client/**`,
  Tailwind/токены, шрифты.
- НЕ трогаем: `app/api/**`, `lib/server/**`, `supabase/**`, `apps/worker`.
- `packages/shared` — контракт типов. Сейчас засеян нами из §6 промпта (бэка ещё нет);
  когда бэкенд появится — он владеет, мы только импортируем. См. PROGRESS → OPEN QUESTIONS.

## Дизайн-токены (§3 промпта — соблюдай строго)
```
--paper   #FFFFFF   фон
--wash    #FBF8F3   тёплый wash
--ink     #0D0C11   заголовки/текст
--ink2    #3A3742
--muted   #7C7886   вторичный
--faint   #B9B5BE
--hair    rgba(13,12,17,.09)  хайрлайны/бордеры
--ember   #FF5A1F   акцент
--ember2  #FF8C3C   акцент (градиент 96deg ember→ember2)
--win     #100E16   тёмная «поверхность приложения» (герой-объект, thumbnails)
--win2    #17141F
--winline rgba(255,255,255,.11)
```
- Display: system/SF (`-apple-system, "SF Pro Display", Inter, system-ui`),
  **tight, lowercase**, `letter-spacing:-.03em`.
- Mono (числа/статусы/тех-лейблы): `"SF Mono", ui-monospace, "JetBrains Mono"`.
  Микролейблы UPPERCASE с трекингом. Заголовки lowercase.
- Радиусы 16–22px, бордеры `--hair`, тень `0 30px 60px -40px rgba(20,12,30,.25)`.
- Белый фон + воздух; тёмное — только сам продукт как герой-объект.
- Ember дозировано (CTA, активный статус, прогресс). Не заливай всё.
- Моушн-мотивы: REC-точка + mono-таймер, ember-скраббер (прогресс рендера),
  ember click-ring на hover/действии. Уважать `prefers-reduced-motion`.

## Фикс багов
- Сначала корневая причина + взгляд на архитектуру. Никаких точечных костылей.
- Фикс трогает 3+ места → проблема в дизайне, предложи рефактор.
- 2 попытки не сработали → стоп, 3 гипотезы в OPEN QUESTIONS, спроси.
- После фикса — проверь смежные модули.

## Рабочий процесс
- После изменений: `pnpm typecheck && pnpm build`, чини сам.
- Не выдумывай API библиотек — проверь типы в node_modules/доках.
- Не выдумывай форму ответа бэка — типы из `packages/shared`. Нет эндпоинта →
  `// TODO(stub)` + пункт в OPEN QUESTIONS, мок помечай явно.
- Не объявляй «готово», не запустив экран.

## Код
- Минимальный diff. Не рефактори соседнее без запроса.
- Перед новым файлом/функцией — проверь, нет ли аналога.
- Иммутабельность: новые объекты, не мутируй. Файлы 200-400 строк (max 800).
- Новые зависимости — только с подтверждением.
